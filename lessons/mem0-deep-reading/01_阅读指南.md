## 阅读策略

### 第一层：骨架理解（30 分钟）
**目标**：不看细节，先记住"两阶段流水线 + 四层记忆 + 三个攻击面"。

### 第二层：架构对标（60 分钟）
**目标**：把论文中的 Mem0 架构映射到你正在写的代码：
- 论文 Phase 1 Extraction → 你的 `extractor.py` + `EXTRACTION_PROMPT`
- 论文 Phase 2 Update → 你的 `daemon.py` 中的 `MemoryUpdater`
- 论文 Top-k 向量检索 → 你的 `memory_search`（目前用 substring，后续换向量）

**关键问题**：你的实现和论文的正式架构有哪些差距？哪些是你已经做得对的？哪些是需要补的？

### 第三层：安全深度（45 分钟）
**目标**：理解三个攻击面，能在面试中讲清楚：
- "Mem0 用 LLM 做 Update 决策，这本身是一个攻击面——如果攻击者诱导 LLM 执行 DELETE 正确记忆，怎么办？"
- "Retrieval 漏洞是最严重的，因为注入的记忆不需要被验证，只需要被检索到。"

### 第四层：扩展实现（90 分钟）
**目标**：对照 Mem0 GitHub 开源代码，理解生产级实现细节。
