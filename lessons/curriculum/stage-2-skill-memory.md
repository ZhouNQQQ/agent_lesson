## 3. 阶段二：Skills开发与记忆系统（2-3周）

### 3.1 学习目标（SMART）

- **具体（Specific）**：掌握自定义Skill开发全流程，深入理解Memory系统两层架构，开发一个带长期记忆的个人助手
- **可衡量（Measurable）**：独立完成3个不同复杂度的自定义Skill（简单/中等/复杂），个人助手能跨会话记住用户偏好
- **可实现（Achievable）**：基于阶段一的平台使用经验，有编程基础的开发者2-3周可完成
- **相关性（Relevant）**：Skill开发是Agent能力扩展的核心方式，记忆系统是Agent"像人"的关键
- **时限（Time-bound）**：3周内完成

### 3.6 常见坑点与避坑指南

| 坑点 | 现象 | 解决方案 |
|------|------|----------|
| SKILL.md格式错误 | Gateway无法识别新Skill | YAML frontmatter必须严格格式，---分隔符不能少 |
| 脚本权限不足 | Skill触发后执行失败 | `chmod +x scripts/*.py` 确保执行权限 |
| 依赖缺失 | 脚本导入模块报错 | 在Skill目录下创建requirements.txt并安装 |
| 记忆未生效 | Agent不记得之前的事 | 检查MEMORY.md格式，确认网关重启后加载 |
| 记忆覆盖 | 手动编辑的记忆被自动覆盖 | 理解记忆写入时机，在Agent空闲时编辑 |
| API Key泄露 | Skill脚本中硬编码密钥 | 使用环境变量或QClaw的密钥管理功能 |

### 3.7 阶段二核心技术深入

**SKILL.md设计哲学**：SKILL.md不仅是一份说明书，它是Agent理解"何时调用、如何调用、期望什么"的核心契约。好的SKILL.md设计遵循以下原则：

1. **描述即契约**：Markdown描述要清晰说明Skill的功能、参数、返回值和边界情况
2. **示例驱动**：提供2-3个完整的调用示例（包括输入和预期输出）
3. **错误明确**：说明所有可能的错误情况和处理方式
4. **权限最小化**：只申请必要的权限，遵循安全最佳实践

**记忆注入的时序模型**：理解记忆何时被注入Agent的上下文窗口至关重要。标准注入时序为：

```
会话开始
  → 加载SOUL.md（人格定义）
  → 加载AGENTS.md（能力定义）
  → 加载USER.md（用户关系）
  → 加载MEMORY.md（长期记忆，按相关性筛选）
  → 加载今日日志（当天会话历史）
  → 加载相关历史日志（向量搜索召回）
  → 开始对话
```

理解这个时序后，你就可以通过精心维护MEMORY.md来影响Agent的行为，也能诊断记忆未生效的问题。

**Mem0两阶段流水线深度理解**：

Mem0的核心是一个两阶段记忆处理流水线：

1. **Extraction阶段**：从对话中提取结构化记忆片段
   - 输入：原始对话文本
   - 处理：LLM分析对话，识别事实、偏好、关系等需要记忆的信息
   - 输出：结构化记忆条目（含类别、重要性、时间戳）

2. **Update阶段**：将提取的记忆合并到长期记忆存储
   - 四种操作：ADD（新增）、UPDATE（更新）、DELETE（删除）、NOOP（无操作）
   - 去重和冲突解决：新记忆与已有记忆对比，决定是更新还是新增
   - 存储写入：最终写入MEMORY.md或向量数据库

理解这个流水线后，你就可以：
- 调试记忆不生效的问题（哪个阶段出错了？）
- 优化记忆提取质量（调整Extraction的Prompt）
- 自定义记忆更新策略（何时更新、如何合并）
