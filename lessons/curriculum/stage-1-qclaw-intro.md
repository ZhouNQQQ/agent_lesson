### 2.6 常见坑点与避坑指南

| 坑点 | 现象 | 解决方案 |
|------|------|----------|
| 端口冲突 | Gateway启动失败，提示18789端口被占用 | `lsof -i :18789`查看占用进程，修改配置或杀掉占用进程 |
| 模型API Key未配置 | 发送消息后无回复或报错 | 在控制台或配置文件中正确填入模型API Key |
| 微信绑定失败 | 扫码后无法连接 | 确保手机和电脑在同一网络，检查防火墙设置 |
| 内存不足 | QClaw运行卡顿 | 关闭不必要的应用，或降低模型参数（如使用轻量级模型） |
| 权限问题（macOS） | 安装时提示无法打开 | 系统设置 -> 隐私与安全 -> 允许 |
| 记忆文件找不到 | 不知道记忆存在哪里 | 查看`~/.openclaw/memory/`目录 |

### 2.7 阶段一深度理解要点

在学习阶段一的过程中，不要仅仅停留在"会用"层面，要深入理解以下核心机制：

**Gateway的工作机制**：Gateway作为入口层，不仅仅是简单的消息转发。它负责会话的生命周期管理、模型路由调度、多渠道消息的归一化处理。当你通过微信发送一条消息时，Gateway会完成：消息接收 -> 格式转换 -> 模型选择 -> 请求发送 -> 响应处理 -> 格式转换 -> 消息推送的完整链路。

**Agent的行为定义三层模型**：
- `SOUL.md`定义Agent的"灵魂"——核心人格、价值观、行为准则
- `AGENTS.md`定义Agent的"能力"——可用工具、Skills、任务范围
- `USER.md`定义Agent的"关系"——对当前用户的认知、偏好、交互历史

这三层是叠加关系：SOUL提供底层人格约束，AGENTS提供能力边界，USER提供个性化上下文。理解这个分层模型，对后续开发带记忆的助手至关重要。
